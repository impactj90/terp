# Absences (/absences)

## Purpose
Allow employees to request time off, view their absence calendar, and track approval status.

## Audience
- Users: Request and track their own absences.
- Admins: Use Admin > Approvals to approve or reject.

## How to use (step-by-step)
1. Open Absences from the main navigation.
2. Click Request Absence (or click a date in the calendar).
3. Choose an Absence Type, date range, and duration (full/half day).
4. Add optional notes and submit.
5. Track status in Your Requests and the calendar.
6. Delete pending requests if needed.

## Data and fields
- Absence type: category, color, paid/unpaid, affects vacation, requires approval.
- Date range: single day or multi-day; half-day only for single-day selections.
- Notes: optional free text.
- Status: pending, approved, rejected (shown in the requests list).

## Where changes take effect / integrations
- Approved absences affect daily values and vacation balances.
- Absences appear in Team Overview and Admin > Approvals.
- Vacation Impact Preview uses the current vacation balance and holidays.

## Troubleshooting
- Overlapping dates are blocked; adjust the range to avoid existing absences.
- If no absence types appear, they must be configured in Admin > Absence Types.
- If the user has no employee record, the page shows a message and actions are disabled.
