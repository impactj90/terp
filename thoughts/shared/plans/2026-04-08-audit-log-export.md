# Implementation Plan: Audit-Log Export (CSV + PDF)

Date: 2026-04-08

## Overview

Add CSV and PDF export capabilities to the existing Audit-Log module. Users with the new `audit_log.export` permission can export filtered audit log entries as UTF-8 CSV (Excel-compatible) or A4-landscape PDF with tenant branding.

## Architecture Decision

**Server-side export via tRPC mutations** (same pattern as DATEV export in `inbound-invoice-datev-export-service.ts`):
- Audit logs can have thousands of entries; server has direct DB access without round-trips
- PDF generation requires `@react-pdf/renderer` which runs server-side
- Returns base64-encoded content; frontend decodes and triggers browser download
- No Supabase Storage needed -- direct base64 return (unlike stocktake PDF which uploads to storage)

Add function `findAllForExport`:

```ts
export async function findAllForExport(
  prisma: PrismaClient,
  tenantId: string,
  params?: AuditLogExportParams,
  limit = 10000
) {
  const where = buildWhere(tenantId, params as AuditLogListParams)
  return prisma.auditLog.findMany({
    where,
    include: auditLogUserInclude,
    orderBy: { performedAt: "desc" },
    take: limit + 1,  // fetch limit+1 to detect overflow
  })
}
```

Fetching `limit + 1` lets the service layer detect when there are more records than the limit and return a clear error.

Also add `countForExport` to check total count before fetching:

```ts
export async function countForExport(
  prisma: PrismaClient,
  tenantId: string,
  params?: AuditLogExportParams
) {
  const where = buildWhere(tenantId, params as AuditLogListParams)
  return prisma.auditLog.count({ where })
}
```

**Note:** The existing `buildWhere` function already handles all filter params. We reuse it by casting `AuditLogExportParams` to `AuditLogListParams` (the page/pageSize fields are optional and simply undefined).

### Step 2.1 -- Create audit-log-export-service.ts

**Constants:**

**Error classes:**

```ts
export class AuditLogExportLimitExceededError extends Error {
  constructor(count: number, limit: number) {
    super(`Export limit exceeded: ${count} entries found, maximum is ${limit}. Please narrow your filters.`)
    this.name = "AuditLogExportLimitExceededError"
  }
}

export class AuditLogExportEmptyError extends Error {
  constructor() {
    super("No audit log entries match the current filters.")
    this.name = "AuditLogExportEmptyError"
  }
}
```

Both error class names end in `Error` which maps correctly in `handleServiceError`:
- `*ValidationError` / `*InvalidError` -> BAD_REQUEST -- but these are not "validation" errors per se
- Better: Use `AuditLogExportLimitExceededError` which `handleServiceError` does not match by name -> falls through to INTERNAL_SERVER_ERROR. So we should use names that map correctly:
  - `AuditLogExportValidationError` for limit exceeded (maps to BAD_REQUEST)
  - `AuditLogExportNotFoundError` for empty results (maps to NOT_FOUND) -- or just return empty, no error

Actually, re-reading `handleServiceError`: it checks `error.name.includes("NotFound")` -> NOT_FOUND, `error.name.includes("Validation") || error.name.includes("Invalid")` -> BAD_REQUEST. So:

```ts
export class AuditLogExportValidationError extends Error {
  constructor(message: string) {
    super(message)
    this.name = "AuditLogExportValidationError"
  }
}
```

Use this single error class for both limit exceeded and empty result cases.

**Shared input type:**

```ts
export interface AuditLogExportInput {
  userId?: string

  entityId?: string

```

**Core data-fetching function (shared by CSV and PDF):**

```ts
async function fetchExportData(
  prisma: PrismaClient,
  tenantId: string,
  input: AuditLogExportInput
) {
  // 1. Count first to check limit
  const total = await repo.countForExport(prisma, tenantId, input)

  if (total === 0) {
    throw new AuditLogExportValidationError(
      "Keine Audit-Protokolleintraege entsprechen den aktuellen Filtern."
    )
  }

  if (total > EXPORT_LIMIT) {
    throw new AuditLogExportValidationError(
      `Export-Limit ueberschritten: ${total} Eintraege gefunden, maximal ${EXPORT_LIMIT} erlaubt. Bitte Filter einschraenken.`
    )
  }

  // 2. Fetch all matching records
  const records = await repo.findAllForExport(prisma, tenantId, input, EXPORT_LIMIT)

  return { records, total }
}
```

**CSV export function:**

```ts
export async function exportCsv(
  prisma: PrismaClient,
  tenantId: string,
  input: AuditLogExportInput,
  audit?: AuditContext
): Promise<{ csv: string; filename: string; count: number }>
```

Implementation:
1. Call `fetchExportData()`
2. Build CSV header row (German labels): `Zeitstempel;Benutzer;Personalnummer;Aktion;Entitaetstyp;Entitaets-ID;Entitaetsname;Aenderungen;IP-Adresse`
3. For each record, build a data row:
   - Timestamp: `formatDateTime(record.performedAt)` -> "08.04.2026 14:30:00"
   - User: `record.user?.displayName ?? "System"`
   - Employee number: from user's employee record if available, or empty string
   - Action: raw action value (not localized -- CSV is for data processing, keep machine-readable values)
   - Entity type: raw entity type value
   - Entity ID: `record.entityId`
   - Entity label: `record.entityName ?? ""`
   - Change details: serialize `record.changes` as condensed one-line JSON diff
   - IP address: `record.ipAddress ?? ""`
4. Quote all cells: `"${cell.replace(/"/g, '""')}"`
5. Join cells with `;`, rows with `\n`
6. Prepend UTF-8 BOM: `\uFEFF`
7. Log the export action to audit log (fire-and-forget)
8. Return `{ csv: bomPrefix + csvContent, filename, count }`

**Changes serialization helper:**

```ts
function formatChanges(changes: unknown): string {
  if (!changes || typeof changes !== "object") return ""
  const entries = Object.entries(changes as Record<string, { old?: unknown; new?: unknown }>)
  if (entries.length === 0) return ""
  return entries
    .map(([field, { old: oldVal, new: newVal }]) => `${field}: ${JSON.stringify(oldVal)} -> ${JSON.stringify(newVal)}`)
    .join("; ")
}
```

**Date formatting helper:**

```ts
function formatDateTime(date: Date | string): string {
  const d = new Date(date)
  return new Intl.DateTimeFormat("de-DE", {
    day: "2-digit", month: "2-digit", year: "numeric",
    hour: "2-digit", minute: "2-digit", second: "2-digit",
  }).format(d)
}
```

**Filename generation:**

```ts
const dateStr = new Date().toISOString().slice(0, 10).replace(/-/g, "")
const filename = `Audit-Log_${dateStr}.csv`
```

**Audit logging the export itself:**

```ts
if (audit) {
  await auditLog
    .log(prisma, {
      tenantId,
      userId: audit.userId,
      action: "export",
      entityType: "audit_log",
      entityId: "batch",
```

Add two new mutations to the router:

```ts
exportCsv: tenantProcedure
  .use(requirePermission(AUDIT_LOG_EXPORT))
  .input(exportInputSchema)
  .mutation(async ({ ctx, input }) => {
    try {
      const { exportCsv } = await import("@/lib/services/audit-log-export-service")
      const audit = {
        userId: ctx.user!.id,
        ipAddress: ctx.ipAddress ?? null,
        userAgent: ctx.userAgent ?? null,
      }
      const result = await exportCsv(ctx.prisma, ctx.tenantId!, input, audit)
      return {
        csv: Buffer.from(result.csv, "utf-8").toString("base64"),
        filename: result.filename,
        count: result.count,
      }
    } catch (err) {
      handleServiceError(err)
    }
  }),

exportPdf: tenantProcedure
  .use(requirePermission(AUDIT_LOG_EXPORT))
  .input(exportInputSchema)
  .mutation(async ({ ctx, input }) => {
    try {
      const { exportPdf } = await import("@/lib/services/audit-log-export-service")
      const audit = {
        userId: ctx.user!.id,
        ipAddress: ctx.ipAddress ?? null,
        userAgent: ctx.userAgent ?? null,
      }
      const result = await exportPdf(ctx.prisma, ctx.tenantId!, input, audit)
      return {
        pdf: result.pdf.toString("base64"),
        filename: result.filename,
        count: result.count,
      }
    } catch (err) {
      handleServiceError(err)
    }
  }),
```

**Dynamic imports:** Both use `await import(...)` to avoid bundling `@react-pdf/renderer` and the export service into every route. This matches the stocktake PDF pattern.

**Note on ctx fields:** Check that `ctx.ipAddress` and `ctx.userAgent` exist in the tRPC context. If not, extract from headers in the procedure:

**Export handler functions:**

```ts
const exportCsvMutation = useExportAuditLogsCsv()
const exportPdfMutation = useExportAuditLogsPdf()

const currentFilters = {
  userId: userId ?? undefined,
  entityType: entityType ?? undefined,
  entityId: entityId || undefined,
  action: action ?? undefined,
  fromDate: fromStr,
  toDate: toStr,
}

const handleExportCsv = async () => {
  try {
    const result = await exportCsvMutation.mutateAsync(currentFilters)
    // Decode base64 and download
    const binaryString = atob(result.csv)
    const bytes = new Uint8Array(binaryString.length)
    for (let i = 0; i < binaryString.length; i++) {
      bytes[i] = binaryString.charCodeAt(i)
    }
    const blob = new Blob([bytes], { type: 'text/csv;charset=utf-8;' })
    triggerDownload(blob, result.filename)
    toast.success(t('export.success', { count: result.count }))
  } catch {
    toast.error(t('export.error'))
  }
}

const handleExportPdf = async () => {
  try {
    const result = await exportPdfMutation.mutateAsync(currentFilters)
    const binaryString = atob(result.pdf)
    const bytes = new Uint8Array(binaryString.length)
    for (let i = 0; i < binaryString.length; i++) {
      bytes[i] = binaryString.charCodeAt(i)
    }
    const blob = new Blob([bytes], { type: 'application/pdf' })
    triggerDownload(blob, result.filename)
    toast.success(t('export.success', { count: result.count }))
  } catch {
    toast.error(t('export.error'))
  }
}

function triggerDownload(blob: Blob, filename: string) {
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')
  a.href = url
  a.download = filename
  document.body.appendChild(a)
  a.click()
  document.body.removeChild(a)
  URL.revokeObjectURL(url)
}
```
